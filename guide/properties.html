# Properties

The component accepts 6 props (one of which is provided for the v-model property):

Name Type Default value/Required
options Array required (it's v-model)
total Number | Null The sum of the value values in the options array
step Number 1
disabled Boolean false
min-value Number 0
max-value Number | Null null
styles Object {}

Something like this would look like a component that uses all the properties that can only be passed to it.

<template>
	<VuePartition
		v-model="options"
		:total="1000"
		:step="1"
		:disabled="isDisabled"
		:min-value="10"
		:max-value="null"
		:styles="partitionStyles"
	/>
</template>

<script>
 import { VuePartition } from '@eneropl/vue-partition';

 export default {
  components: { VuePartition },
  data() {
    return {
      // options
    }
  }
 }
</script>

# Options

This property accepts an array of objects, each of which represents a separate section. The property has many options for configuring the behavior and visual appearance of your section, so this section is going to be long :3

Below is an object with properties that you can use to define a section (? indicates that the property is optional.):

{
  value: 100,
  minValue?: 10,
  maxValue?: 200,
  editable?: true,
  disabled?: false,
  focused?: false,
  styles: {},
}

# value

This property defines the value of your section. Each section (of the value value) in the component's view is converted to the width of the component's container (the default component width is 500px, so a value of 100 will be converted to 25px, depending on the number of sections, etc.), so you don't have to worry about what length value you are using.

By default, if you don't specify a value property, it will be 0.

Value validation

The component does not validate your values. Therefore, if you set the total property to be less than the sum of all value options, then this may cause the component to work incorrectly.

# minValue

This property specifies the minimum allowed value that can be specified for the value property when editing your object. This component defines the global value "min-value", but if we specify this property in the object, then it will have a higher priority.

By default, this property is 0. This means that when editing a partition, the current section may disappear if its value when changed is <= 0.

Priority of minValue

The property takes precedence over the component's :min-value property.

# maxValue

A property that defines the maximum value that an element can only reach. If, while editing a partition, the current section reaches its maximum value, then it will stop being edited. Relative to the global "max-value" property defined for the partition component, that particular property will take precedence.

By default, this property is null. This means that the object does not have a maximum limit.

Priority of maxValue

The property takes precedence over the component's :max-value property.

# editable

Specifies the state of an object that is available as editable. If the property is true, then the object can change its value and affect other sections depending on the conditions set.

Ignoring property

This property will be ignored if the disabled property is defined for the object.

# disabled

Specifies the state of the section. If a section is disabled ("disabled: true"), then no editing condition will affect this section, it will keep its value unchanged.

# focused

Specifies the ability to set the default section as the section that will first accept changes to the main editable section. This means that wherever you define an editable object (even far, far away from the "focused: true" setting), it will first work on the focused section as long as its modifier conditions return true.

# styles

An object that represents customization settings and has the following editing properties (property values are provided as default values if you don't specify them).

Styles properties

You can read more about style properties here

# Total

The total value, which is defined as the sum of all value passed in options. If the value is not specified, then the component will independently calculate the sum of all passed value in the array of options objects.

Validation

The total property cannot be equal when the options objects have value values ​​that sum to more than total.

# Step

Specifying a value for this property will determine how many times your sections change. The default is 1, which means your changes to all sections will always be a multiple of 1 (will also work with any other integer values).

# Disabled

Completely blocks the component's handlers. If you have editable sections, then they are no longer editable.

# MinValue & MaxValue

Global setting of the minimum/maximum value for each section. Each section will comply with the set condition, except for those sections that do not define their own minValue and\or maxValue internally.

Ignoring property

This property will be ignored if the minValue or maxValue property is defined for the object of options.
