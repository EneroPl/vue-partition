# Styles

A styles object that you can set globally for a component, as well as locally on a per-section basis with the styles property. The style setting priority will be as follows:

  • Initialize styles property in object of options;
  • Inititalize and\or merge styles property of component styles;
  • Initialize default styles for not setted properties.

Below is an object with properties that the package provides for styling elements:

{
	background: 'transparent',
	control: {
		background: '#FFFFFF',
		borderWidth: 1,
		borderStyle: 'solid',
		borderColor: 'transparent'
	},
	arrows: {
		size: 10,
		background: '#FFFFFF',
		borderWidth: 1,
		borderStyle: 'solid',
		borderColor: 'transparent',
		prev: {
			background: '#FFFFFF',
			borderWidth: 1,
			borderStyle: 'solid',
			borderColor: 'transparent'
		},
		next: {
			background: '#FFFFFF',
			borderWidth: 1,
			borderStyle: 'solid',
			borderColor: 'transparent'
		}
	}
}

# Control styles

Styles that define the controller element for the section being edited.

# Arrows styles

Styles that define the arrow element for the editable section. As you can see, the arrows property has 2 more objects defined: prev & next. This means that you have the opportunity to individually define each of the arrows.

General arrows styles

You can define general styles in the arrows property, and if you need to override them on a particular arrow, then you can specify arrows.prev (for example) with the styles you need. Just be aware that the arrows.prev & arrows.next styles take precedence.
